## Findings

### Fixed

1. **HIGH - Duplicated STATUS_STYLES constant** (LegislativeIndex.tsx, Legislative.tsx)
   - Category: DRY violation
   - Both pages defined identical STATUS_STYLES objects
   - **Fix:** Extracted to `shared/procedureStatus.ts`, imported in both

2. **MEDIUM - Unused import** (LegislativeIndex.tsx:2)
   - Category: Code quality
   - `ExternalLink` imported from lucide-react but never used
   - **Fix:** Removed

3. **MEDIUM - Unused import** (committeeRouter.ts)
   - Category: Code quality
   - `documentChunks` imported but not used (fixed in earlier commit)
   - **Fix:** Already removed

4. **MEDIUM - Fragile sql.join pattern** (procedureRouter.ts)
   - Category: Code quality
   - Used `sql.join` for WHERE conditions instead of Drizzle's `and()`
   - **Fix:** Refactored to `and(...conditions)` (fixed in earlier commit)

### Reviewed and accepted (no action needed)

5. **LOW - SQL LIKE pattern for committee membership** (committeeRouter.ts:31, 87)
   - The `LIKE '%ITRE%'` pattern on JSON cast to text could theoretically match substrings. However, committee abbreviations (ITRE, IMCO, etc.) are unique enough that false matches are effectively impossible with the current dataset. The abbreviation comes from our own config, not user input. Acceptable.

6. **LOW - ILIKE title search for committee votes** (committeeRouter.ts:148)
   - Votes are matched by searching titles for committee abbreviation or shortName. This is a best-effort heuristic since votingRecord doesn't have reliable committee FK. The ILIKE values come from our config after slug lookup validation, so no injection risk. Acceptable for MVP.

7. **LOW - No tests** for new endpoints/components
   - Project has no test infrastructure set up yet (vitest config exists but no test files). This is a known gap for the whole project, not specific to this change. Not addressing now.

8. **LOW - Speeches matched by name string** (mepRouter.ts:67)
   - `speakerName` equality match could miss MEPs with name variations. This matches the data as ingested (speaker names from CRE transcripts). Acceptable.
